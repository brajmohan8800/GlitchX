How I approach testing

The Software Testing Life Cycle I actually follow, step by step, on every project.

01
Requirement Analysis
Understand what a feature is supposed to do — and just as importantly, what it's not.
02
Test Planning
Decide scope, tools and priorities before writing a single test case.
03
Test Case Design
Write clear, repeatable cases — happy paths and edge cases both.
04
Test Execution
Run cases manually or via automation, and log everything that happens.
05
Defect Reporting
File clean, reproducible bug reports developers actually want to read.
06
Regression & Closure
Re-test fixes, confirm nothing else broke, then sign off.
